**Onboarding Update – Rules System For Long Term Investors**

We need the rules system to adapt depending on the user's Primary Trading Style so it feels natural for both traders and investors.

---

**Rules Naming Based on Trading Style**

If user selects:

Day Trader or Swing Trader:
Section should be called:

**Trading Rules**

If user selects:

Long Term Hold:
Rename this section to:

**Investing Principles**

(Important: This is only a wording change. It should still use the same backend rules system.)

---

**Minimum Rules Requirement**

Day Traders and Swing Traders:
Minimum **5 rules required**

Long Term Hold users:
Minimum **3 principles required**

Goal:
Reduce friction for investors while still enforcing discipline structure.

---

**Helper Text Change**

For traders:

"Add the rules you must follow to stay disciplined."

For investors:

"Add the principles you follow to avoid emotional investing decisions."

---

**Quick Add Templates (Important UX improvement)**

For Long Term Hold users, provide quick-add buttons to make onboarding easier.

Examples:

[ Never panic sell ]
[ Only invest long term ]
[ Diversify positions ]
[ Follow my investment thesis ]
[ Ignore short term noise ]
[ Only invest in companies I understand ]
[ Never chase hype ]

When tapped:
Automatically adds as a principle that can be edited.

User must still be able to add custom principles.

---

**Category Adjustment For Investors**

Rule categories should slightly adapt wording:

Instead of:

Psychology
Entry
Risk
Exit
Discipline

Long Term Hold should see:

Psychology
Buying Criteria
Risk Management
Selling Criteria
Discipline

(Again, only label changes, same backend structure.)

---

**Important Functional Requirement**

Rules/Principles must still:

• Show in RevengeX when user clicks "Read My Rules"
• Show in Edit Rules screen
• Be stored in same database structure
• Be categorized automatically
• Allow Critical Rule ⭐ marking

Do NOT create separate systems.

Only adjust labels based on trading_style.

---

**Design Goal**

Investors should feel:

"This app understands how I invest."

Not:

"This is a day trading app I'm trying to force myself into."

The experience should feel tailored through wording only, not separate flows.
